Omega SFM Development

As part of my 3rd Year Placement I worked as a Software Engineer for WesternGeco, a subsidiary of Schlumberger LTD group, a company which specializes in oilfield services with offices in Gatwick. I was assigned to the REMS (Research, Engineering, Manufacturing, and Sustaining) team, composed of software developers and geophysicists responsible for the development and maintenance of a Geophysics Data Processing software called Omega.

View Interim Report

Wine Rating Predictor

A project for EE3-23 Introduction To Machine Learning where I worked on a machine learning model to train models using datasets of red and white variants of the Portuguese ”Vinho Verde” wine in order to predict the wine rating as accurately as possible. The dataset contains 11 inputs of objective tests (e.g. PH values) and one target rating, which is based on sensory data(median of at least 3 evaluations made by wine experts). The project explored different approaches to classification algorithms and their advantages/disadvantages in the context of this dataset.

View Project Report

SwimTime: Swimming Companion

As part of the Third-Year Embedded Systems course we were required to design, using a list of available sensor, an IoT device which is connected to a online platform using the MQTT protocol for lightweight data transfer. We came up with SwimTime, an device which using an ultra-sonic distance sensor would act as a swimming companion by measuring lap times and speed. This device would then transfer the data to a online Dashboard where the user would be able to monitor his/her progress in the swimming pool. The device was designed as a proof-of-concept and implemented on a breadboard.

Advanced Signal Processing

A coursework assignment for module EE3-08 Advanced Signal Processing, where we developed signal processing skills and algorithms using MATLAB scripts and functions. The course was aimed at understanding the basics of real-worlds signal processing where we were required, among other things, to work with ECG signals in order to understand tha challenges faced by the industry in processing noisy signals.

View Project Report

WiSP: Wireless Charge Sharing device

My Second year group project consisted in developing a charge sharing device with an associated platform. The project aimed to create an environment in which users can and are incentivised to share charge between mobile devices. The project envisions the implementation of this as a wireless power transmitter/receiver circuit integrated with the mobile device which is controlled by an app that manages geo-matching and share requests between users, giving customers the opportunity to charge their phones wherever another mobile device is available.
